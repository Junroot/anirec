1\. Executive Summary

1.1 Product Vision

> **⚠ v1.1 변경:** v1.0 대비 제품 포지션이 ‘종합 플랫폼’에서 ‘MAL 연동 인덱스 + AI 추천’으로 변경되었습니다.

AniRec은 애니메이션을 탐색하고 개인 평점을 기록하면, 그 데이터와 MyAnimeList(MAL) 데이터를 결합하여 AI가 취향에 맞는 애니메이션을 추천해주는 인덱스 역할의 플랫폼입니다.

핵심 컨셉

- AniRec은 애니메이션을 찾아주는 ‘인덱스’ 역할에 집중합니다.

- 애니메이션의 상세 정보(줄거리, 캐릭터, 에피소드 목록 등)는 MAL에서 조회합니다.

- 탐색 화면의 평점은 MAL 평점을 표시하여 초기부터 신뢰성 있는 데이터를 제공합니다.

- 사용자가 남기는 평점은 본인만 볼 수 있는 개인 데이터이며, 추천 엔진의 입력값으로 사용됩니다.

1.2 Problem Statement

- 기존 플랫폼은 평점 데이터를 활용한 개인화 추천이 부족합니다.

- MAL/AniList에는 풍부한 데이터가 있지만, ‘내 취향’을 반영한 추천은 없습니다.

- 새로운 애니메이션을 찾는 과정이 비효율적입니다.

1.3 Target Users

- 애니메이션을 정기적으로 시청하며 새로운 작품을 찾고 싶어하는 애니메이션 팔 (18–35세)

- 취향에 맞는 작품을 효율적으로 발견하고 싶은 사용자

- MAL을 주로 사용하지만 더 나은 추천이 필요한 사용자

2\. Product Principles

> **⚠ v1.1 변경:** v1.1에서 신규 추가된 섹션입니다. 제품 의사결정의 기준이 되는 원칙을 명시합니다.

2.1 인덱스 역할에 집중

AniRec은 애니메이션을 ‘찾아주는’ 것에 집중합니다. 애니메이션의 상세 정보(줄거리, 캐릭터, 에피소드 목록, 리뷰 등)는 MAL이라는 검증된 플랫폼에 위임하여 개발 범위를 최소화하고, 핵심 가치인 ‘추천’에 리소스를 집중합니다.

2.2 MAL 데이터 우선

탐색 화면의 평점은 MAL 평점을 표시합니다. 자체 사용자 평점 데이터가 충분히 축적되기 전까지, 수백만 건의 MAL 평점을 활용하여 초기부터 신뢰성 있는 평점 데이터를 제공합니다.

2.3 개인 평점 = 추천 입력값

사용자가 남기는 평점은 탐색 화면에 노출되지 않습니다. 본인만 확인 가능한 개인 데이터이며, 오직 추천 엔진의 입력값으로만 활용됩니다. 이를 통해 사용자는 타인의 시선을 의식하지 않고 솜직하게 평점을 남길 수 있습니다.

2.4 기능 최소주의

한줄평, 자체 상세 페이지 등 MAL과 중복되는 기능은 구현하지 않습니다. 애니메이션 카드를 클릭하면 MAL 상세 페이지로 이동하여 기존 생태계와 공존합니다.

4.2 애니메이션 탐색 (Browse & Search)

> **⚠ v1.1 변경:** 탐색 화면의 평점은 MAL 평점을 표시합니다. 사용자 평점은 탐색 화면에 노출되지 않습니다. 카드 클릭 시 MAL 상세 페이지로 이동합니다.

4.2.1 기능 설명

MAL API에서 애니메이션 목록과 평점을 가져와 카드 형태로 표시합니다. 애니메이션 카드를 클릭하면 자체 상세 페이지 없이 MAL 해당 작품 페이지로 새 탭에서 이동합니다.

4.2.2 요구사항

- 텍스트 검색: 제목 기반 실시간 검색 (Ctrl+S Quick Search 모달)

- 필터링: 장르(Genre), 연도(Year), 시즌(Season), 포맷(TV/Movie/OVA)

- 정렬: MAL 평점순, 인기순, 최신순, 이름순

4.2.3 카드 UI 구성

| **제목**          | 애니메이션 제목 (원제/영어)                             |

4.3 평점 시스템 (Personal Rating)

> **⚠ v1.1 변경:** 평점은 공개되지 않으며 본인만 확인 가능합니다. 한줄평 기능은 제거되었습니다.

4.3.1 기능 설명

사용자가 애니메이션에 개인 평점을 부여하고 시청 상태를 관리합니다. 이 데이터는 본인만 볼 수 있으며, 추천 엔진의 입력값으로만 활용됩니다.

4.3.2 요구사항

- 평점 입력: 1–10점 스케일 (별점 클릭 또는 슬라이더)

- 시청 상태: Watching / Completed / Plan to Watch / Dropped / On Hold

- 평점 수정/삭제 가능

- 평점 입력 진입점: 카드 호버 시 \[평점 남기기\] 버튼 또는 내 평점 페이지

4.3.3 평점 공개 정책

|                  |                  |                  |
|------------------|------------------|------------------|
| **항목**         | **공개 여부**    | **사용처**       |
| 내 평점 (1–10점) | 비공개 (본인만)  | 추천 엔진 입력값 |
| 시청 상태        | 비공개 (본인만)  | 추천 필터링 조건 |
| MAL 평점         | 공개 (탐색 화면) | 카드 표시용      |

4.4 추천 시스템 (Recommendation Engine)

> **⚠ v1.1 변경:** v1.2: 추천 알고리즘이 MAL 사용자 평점 기반 클러스터링 방식으로 변경되었습니다. MAL의 수백만 사용자 평점 데이터를 활용하여 취향 그룹을 형성하고, 동일 그룹 내 고평점 작품을 추천합니다.

4.4.1 추천 알고리즘 개요

MAL에는 수백만 명의 사용자가 남긴 평점 데이터가 축적되어 있습니다. 이 데이터를 활용하여 MAL 사용자들을 평점 패턴이 비슷한 그룹으로 묶고, AniRec 사용자의 개인 평점을 기반으로 가장 비슷한 그룹을 찾아, 해당 그룹 내에서 고평점을 받았지만 아직 사용자가 보지 않은 작품을 추천합니다.

4.4.2 알고리즘 파이프라인

|          |                   |                                                                                                                              |                     |
|----------|-------------------|------------------------------------------------------------------------------------------------------------------------------|---------------------|
| **Step** | **단계**          | **상세 로직**                                                                                                                | **실행 주기**       |
| **1**    | MAL 평점 수집     | Jikan API로 MAL 사용자별 평점 리스트를 수집하여 User-Anime 평점 매트릭스 구성                                                | 주 1회 배치         |
| **2**    | 사용자 클러스터링 | K-Means 또는 DBSCAN 알고리즘으로 MAL 사용자들을 평점 패턴 기반 취향 그룹으로 분류 (ex: 액션 러버, 드라마 팔, 이세계 덕후 등) | 주 1회 배치         |
| **3**    | 사용자 매칭       | AniRec 사용자의 개인 평점을 MAL 클러스터와 비교하여 가장 유사한 그룹에 배정 (코사인 유사도 기반)                             | 평점 변경 시 실시간 |
| **4**    | 추천 생성         | 매칭된 그룹 내 MAL 사용자들이 높은 평점을 준 애니 중, AniRec 사용자가 아직 보지 않은(평점 미입력) 작품을 스코어링하여 추천   | 평점 변경 시 실시간 |
| **5**    | 피드백 반영       | 추천 결과에 대한 \[👍\]/\[👎\] 피드백을 반영하여 그룹 매칭 가중치 조정                                                       | 실시간              |

4.4.3 추천 스코어링 공식

추천 스코어(anime_i) = 그룹 내 해당 애니 평균 평점 × W1 + 그룹 내 해당 애니 평가자 비율 × W2 + MAL 전체 평점 × W3 − 이미 본 작품 페널티. 그룹 내 평균 평점이 높고, 많은 그룹 구성원이 평가한 작품일수록 높은 스코어를 받습니다. 가중치 W1–W3는 A/B 테스트로 최적화합니다.

4.4.4 클러스터 예시

|               |                                    |                                     |
|---------------|------------------------------------|-------------------------------------|
| **클러스터**  | **특징**                           | **대표 고평점 애니**                |
| **Cluster A** | Action + Shounen 고평가 그룹       | Jujutsu Kaisen, Demon Slayer, MHA   |
| **Cluster B** | Drama + Slice of Life 선호 그룹    | Frieren, Violet Evergarden, 3-gatsu |
| **Cluster C** | Isekai + Fantasy 덕후 그룹         | Re:Zero, Mushoku Tensei, Slime      |
| **Cluster D** | Psychological + Thriller 선호 그룹 | Death Note, Monster, Steins;Gate    |

*예시: AniRec 사용자가 Frieren에 10점, Violet Evergarden에 9점을 남기면 → Cluster B에 매칭 → Cluster B의 MAL 사용자들이 높게 평가한 3-gatsu no Lion, Anohana 등을 추천*

4.4.5 추천 UI 구성

- 홈 페이지 “나를 위한 추천” 섹션: 추천 카드 4개 + 추천 이유 + \[👍\]/\[👎\] 피드백

- 추천 전용 페이지: 취향 그룹 정보 + 그룹 내 추천 리스트 + 오늘의 추천

- 추천 이유 예시: “당신과 비슷한 취향 그룹의 93%가 8점 이상을 준 작품”

4.5 내 통계 & 평점 관리

4.5.1 내 평점 목록 페이지

내가 평점을 남긴 모든 애니메이션을 목록으로 볼 수 있습니다. 시청 상태별 필터링, 평점순 정렬이 가능합니다.

- 요약 카드: 시청 완료 수, 시청 중, 시청 예정, 평균 평점

- 목록: 애니 제목 + 내 평점 + 시청 상태 + MAL 평점 비교

- 평점 수정/삭제 가능

4.5.2 취향 분석 대시보드

- 장르별 선호도 바 차트 (평균 점수 + 작품 수)

- 평점 분포 히스토그램

- 선호 제작사 TOP 5

- 월별 시청 히스토리

| **공개 평점 표시**   | 초기 데이터 부족. 솜직한 평점 유도 | MAL 평점을 탐색 화면에 표시         |
| **평점 분포 차트**   | 자체 평점 비공개로 의미 없음       | 내 평점 분포는 통계 페이지에서 확인 |
| **포럼 / 소셜**      | MVP 범위 외. 추후 검토             | Phase 4에서 재검토                  |

6\. Information Architecture

6.1 사이트맵 & 라우팅

> **⚠ v1.1 변경:** /anime/:id 상세 페이지가 제거되고, 카드 클릭 시 MAL로 외부 이동합니다.

|                |                  |          |                         |
|----------------|------------------|----------|-------------------------|
| **경로**       | **페이지**       | **인증** | **비고**                |
| /              | 홈 페이지        | ✕        | MAL 평점 표시           |
| /login         | 로그인           | ✕        |                         |
| /signup        | 회원가입         | ✕        |                         |
| /search/anime  | 애니 검색/탐색   | ✕        | MAL 평점 표시           |
| /recommend     | 추천 전용 페이지 | ✓        | MAL + 개인평점 기반     |
| /my-ratings    | 내 평점 목록     | ✓        | 비공개 (본인만)         |
| /stats         | 통계 대시보드    | ✓        | 비공개 (본인만)         |
| (external) MAL | 애니 상세 페이지 | ―        | 카드 클릭 시 새 탭 이동 |

6.2 페이지 플로우

회원가입 → 로그인 → 홈 (추천 + Trending) → 애니 탐색 → 카드 호버에서 평점 남기기 또는 카드 클릭으로 MAL 이동 → 평점 데이터 축적 → MAL 데이터 + 개인 평점으로 추천 엔진 분석 → 추천 생성 → 추천 피드백 → 추천 개선. 이 순환 구조를 통해 사용자가 평점을 남길수록 추천 정밀도가 향상됩니다.

7\. Technical Architecture

7.1 기술 스택

> **⚠ v1.1 변경:** v1.2: 백엔드가 Spring Boot + Kotlin으로 변경되었습니다. 추천 엔진은 Python을 유지하며, Spring Boot에서 내부 API로 호출합니다.

|                 |                                         |                                             |
|-----------------|-----------------------------------------|---------------------------------------------|
| **영역**        | **기술**                                | **선정 이유**                               |
| **프론트엔드**  | React + TypeScript + Tailwind CSS       | 컴포넌트 UI + 타입 안정성 + 유틸리티 CSS    |
| **백엔드**      | Spring Boot 3 + Kotlin + Spring WebFlux | 타입 안정성, 코루틴 기반 비동기, JVM 생태계 |
| **DB**          | MySQL 8 + Redis                         | 개인 평점 저장 + MAL 데이터/추천 결과 캐싱  |
| **ORM**         | Spring Data JPA + QueryDSL              | 타입 안전 쿼리 + MySQL 최적화 쿼리          |
| **추천 엔진**   | Python (scikit-learn) — 내부 API        | K-Means 클러스터링, Spring에서 REST 호출    |
| **애니 데이터** | MAL API (Jikan v4)                      | 무료, 풍부한 데이터, 사용자 평점 포함       |
| **인증**        | Spring Security + JWT + OAuth 2.0       | Spring 네이티브 보안 + 소셜 로그인          |
| **빌드/배포**   | Gradle (Kotlin DSL) + Docker + AWS      | Kotlin 친화적 빌드 + 컨테이너 배포          |
| **테스트**      | JUnit 5 + MockK + Testcontainers        | Kotlin 친화적 Mock + 통합 테스트            |

7.2 시스템 아키텍처

Client (React SPA) → API Gateway (Spring Cloud Gateway) → Spring Boot 애플리케이션 \[Auth Module (Spring Security + JWT/OAuth) + Anime Module (MAL Jikan API 연동 + Redis 캐싱) + Rating Module (MySQL)\] → Recommendation Service (Python, K-Means 클러스터링 엔진, Spring에서 내부 REST API로 호출). Kotlin 코루틴과 WebFlux를 활용하여 MAL API 호출 및 추천 엔진 연동을 비동기로 처리합니다.

7.3 데이터 모델

> **⚠ v1.1 변경:** v1.2: MalUserRating(클러스터링 원본 데이터)과 UserCluster(사용자↔클러스터 매칭) 엔티티가 추가되었습니다.

|                    |                                                  |                     |                             |
|--------------------|--------------------------------------------------|---------------------|-----------------------------|
| **엔티티**         | **주요 필드**                                    | **관계**            | **설명**                    |
| **User**           | id, email, name, avatar, prefs                   | 1:N Rating          | 사용자 계정                 |
| **AnimeCache**     | mal_id, title, score, genres, image_url, mal_url | 1:N Rating          | MAL 데이터 캐시             |
| **Rating**         | id, user_id, mal_id, score, status               | N:1 User, N:1 Cache | 개인 평점 (비공개)          |
| **Recommendation** | id, user_id, mal_id, score, reason, type         | N:1 User            | 추천 결과 캐시              |
| **Feedback**       | id, user_id, rec_id, is_positive                 | N:1 Recommendation  | 추천 피드백                 |
| **MalUserRating**  | mal_user_id, mal_id, score                       | N:N (User↔Anime)    | MAL 사용자 평점 (배치 수집) |
| **UserCluster**    | cluster_id, user_id, centroid_vector, matched_at | N:1 User            | AniRec 사용자↔클러스터 매칭 |

Out of Scope

9\. Risks & Mitigations

|        |                                           |            |                                                           |          |
|--------|-------------------------------------------|------------|-----------------------------------------------------------|----------|
| **ID** | **리스크**                                | **영향도** | **완화 전략**                                             | **담당** |
| **R1** | MAL API (Jikan) Rate Limit 또는 장애      | 높음       | Redis 캐싱 (TTL 24h) + 로컬 DB 펴백 + Rate Limit 준수     | Backend  |
| **R2** | 콜드 스타트 — 초기 개인 평점 부족         | 높음       | 온보딩 시 최소 10개 평점 유도 + MAL 데이터 기반 초기 추천 | Product  |
| **R3** | 추천 편향 — 특정 장르 반복 추천           | 중간       | “오늘의 추천” 새 장르 도전 + 다양성 가중치                | ML       |
| **R4** | 사용자 이탈 — 평점만 남기고 복귀하지 않음 | 높음       | 평점 누적에 따른 추천 품질 향상 시각화 + 간편 평점 UI     | Design   |
| **R5** | MAL 페이지 이동 후 복귀율 저하            | 중간       | 새 탭 열기로 원래 탭 유지 + 평점 입력을 AniRec에서 처리   | Frontend |
